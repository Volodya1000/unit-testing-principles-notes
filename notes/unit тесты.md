
# Что я узнал о том как в [[техосмотр]] относятся к unit тестам.
- Тимофей и Александр Николаевич рассказали, что у них есть unit тесты но их очень мало
- покрыты функции шифрования 
- функция в аппаратной интеллектуальной системе, которая принимает на вход номер машины и определяет корректный он или нет. Очевидно это делается с помощью регулярных выражений. 
- я сделал вывод, что тестами у них покрыты функции, для которых важно проверить, что выход корректный для каждого из входов и при этом вариантов входов много 
- Тимофей сказал, что он бы хотел иметь больше интеграционных тестов, особенно тех которые связаны с базой данных 
- Также Тимофей считает одинаково важными тесты UI  и бэкэнда. Хотя не те не другие особо не пишет 
- Тимофей считает важным делать тесты в UI, которые проверяют, что ui успешно получает ответ на api запросы к бэкэнду. Тоесть тесты соответствия схемы api на фронте и бэке
